# スケジュールと検証計画

## 1. 実行順スケジュール（短期）

### フェーズA（同日）
1. カラースキーム切替UI追加
2. 設定保存復元追加
3. 静的検証（py_compile, ruff）

### フェーズB（同日〜翌日）
1. プロモード限定メタデータ折りたたみパネル追加
2. 画像選択連動更新
3. 既存テスト実行

### フェーズC（仕上げ）
1. Windows実機で見た目確認
2. EXEビルド確認
3. 操作確認（事務員向け導線）

## 2. 検証観点マトリクス
1. カラースキーム切替
   - 条件: System/Light/Dark
   - 期待: 即時反映、UI崩れなし
2. 復元
   - 条件: 再起動
   - 期待: 最終選択が復元
3. プロモード限定表示
   - 条件: 簡易/プロ切替
   - 期待: 簡易で非表示、プロで表示
4. メタデータ有無
   - 条件: EXIFあり画像 / EXIFなし画像
   - 期待: 表示内容が正しい
5. 回帰
   - 条件: 保存・一括保存・EXIF差分
   - 期待: 既存挙動維持

## 3. ロールバック指針
- UI追加で不具合が出た場合、まず表示パネルのみ切り戻し、保存パイプライン側には手を入れない。
- 仕様分割により、カラースキーム切替とメタデータ表示は独立して巻き戻せる構成を維持する。

## 4. 判断ポイント
1. まずカラースキームだけ先行リリースするか。
2. メタデータ表示を同一リリースで含めるか。
3. メタデータ項目を第一段階（要点）で止めるか、拡張するか。
