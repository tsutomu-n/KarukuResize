KarukuResize UI/UX 改善モックアップ

専門家チーム議論に基づく Before / After 比較。各セクションで現状の問題と改善案を視覚的に示す。

1. ボタン階層の導入

すべて同じ青色 → 4段階の視覚的階層で操作の重要度を伝える

BEFORE すべて同じ見た目のボタン
AFTER 4段階の視覚的階層
Primary — 主要アクション（保存系）
Secondary — 補助アクション（非破壊的）
Tertiary — 設定・管理系
Destructive — 破壊的操作
改善理由: Material Design / Apple HIG いずれも、ボタンの視覚的重みで操作の重要度を伝えることを推奨。ユーザーは色と形から「次に何を押すべきか」を直感的に判断できる。

2. ツールバーの整理

13+要素の過密ツールバー → 主動線のみの簡潔なバーに

BEFORE 13+要素が1行に混在
プリセット
px
現在設定: プロモード / テーマ ダーク / 形式 JPEG / 品質 85 / EXIF 保持 / GPS削除 OFF / ドライラン OFF
AFTER 主動線のみ — 7要素以下
px
設定: 品質 95 ドライラン ON JPEG
改善理由: Hick's Law — 選択肢が増えるほど意思決定時間は対数的に増加。ツールバーを7要素以下に抑え、主動線（選択→設定→保存）と副動線（プリセット・設定）を視覚的に分離。設定サマリーは「変更された項目のみ」をチップ表示。
