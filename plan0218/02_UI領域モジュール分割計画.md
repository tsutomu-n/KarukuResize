# 02: UI領域モジュール分割計画

## 目的
- 画面領域ごとにUI構築責務を分割し、再利用性と保守性を上げる。

## 分割ターゲット
- `topbar`（既存: `ui_topbar.py`）
- `settings dialog`（既存: `ui_settings_dialog.py`）
- 追加分割候補:
  - `ui_file_list_panel.py`
  - `ui_preview_panel.py`
  - `ui_metadata_panel.py`
  - `ui_statusbar.py`
  - `ui_detail_controls.py`（Pro詳細領域）

## 実装ステップ
1. 領域単位で「状態入力」「コールバック」「生成Widget参照」を定義する。
2. 各モジュールは `build_*()` と `apply_*()` を基本APIに統一する。
3. `gui_app.py` は `build` 呼び出しと状態反映ディスパッチだけにする。
4. Pro ON/OFFで出し分ける領域は、各モジュールの `apply_mode()` に集約する。
5. `pack/grid` 混在を親コンテナ単位で禁止し、領域ごとにルール固定する。

## 完了条件
- 各主要領域に対応する専用モジュールが存在する。
- `gui_app.py` から領域内部レイアウト詳細が消えている。
- UI差分修正時に、対象領域モジュールだけを編集すれば済む。

## 第三者への実行指示
- 領域ごとにPRを分割し、1PR=1領域を基本とする。
- 受け渡しインターフェースは `dataclass` で明示し、暗黙依存を作らない。
